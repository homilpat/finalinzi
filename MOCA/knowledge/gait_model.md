# 보행 평가 모델

최종 보행 모델은 `gait_daily_clinical_3feat.joblib`이다. 스마트폰을 등쪽 허리 또는 허리 가까운 위치에 고정하고 20초 이상 평소처럼 걸은 CSV를 입력으로 사용한다.

최종 라벨은 낙상이력 자체가 아니라 임상 운동기능 저하를 선별하는 확장 OR 기준이다. 기준은 `TUG >= 12s OR FSST >= 15s OR BERG < 52 OR DGI <= 19 OR base_velocity < 1.0 m/s OR s3_velocity < 1.0 m/s`이다.

최종 입력 피처는 acc-only 3개다. `v_jerk_rms_median`은 수직 jerk RMS의 대표 수준, `v_jerk_rms_iqr`은 보행 중 jerk 변동성, `v_harmonic_ratio_iqr`은 수직 리듬 일관성 변동성을 본다. 보행속도는 라벨에만 사용하고 모델 입력 피처에는 넣지 않아 직접 데이터누수를 피한다.

서비스 threshold는 `0.50`으로 고정한다. 선별 목적상 sensitivity를 0.8 이상으로 유지하는 쪽을 우선했다. 값이 없거나 NaN이 발생하면 예전 모델로 fallback하지 않고 다시 측정하도록 안내한다.

# 보행 측정 안내

사용자는 스마트폰을 허리에 고정하고 평소처럼 20초 이상 걸으면 된다. 방향전환이나 자연스러운 회전은 가능하지만, 뛰거나 폰을 손에 들고 흔드는 방식은 피한다.

데이터가 부족하거나 컬럼이 맞지 않거나 움직임이 거의 없으면 보행 피처가 계산되지 않을 수 있다. 이 경우 "보행 데이터가 충분하지 않아 평가할 수 없습니다. 스마트폰을 허리에 고정하고 평소처럼 20초 이상 걸어 다시 측정해 주세요."라고 안내한다.
